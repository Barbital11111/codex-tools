## 更新日志
- v1.9.0
  1. 新增 Codex token 用量统计，读取本地 Codex 会话记录并展示最近会话、24h、7d、30d token 消耗
  2. 账号页新增隐藏账号信息开关，可隐藏普通账号邮箱、API 接口、模型、Provider 与余额等敏感信息，同时保留自定义 API 名称便于识别
  3. 回退并移除 MiMo 专项兼容尝试，继续保持通用 OpenAI 兼容 API profile 的稳定写入方式
  4. 重写本地打包脚本收束逻辑，生成安装包后会自动识别产物并结束编译进程，减少打包完成后长时间挂起
  5. 调整发布流程与版本资产生成方式，确保安装包、签名文件与 `latest.json` 发布链路更稳定
- v1.8.8
  1. 修复上下文窗口设置写入逻辑：默认会删除相关配置项，点击“应用”后才真正写入 active `.codex/config.toml`
  2. 调整上下文窗口规则：预设仅保留 `默认 / 400K`，手动输入范围改为 `272K-1000K`，并在切换并启动时先应用实例 profile 再重写当前 config
  3. 调整自动压缩阈值规则：上下文窗口会按 `95%` 写入 `model_auto_compact_token_limit`，并且不再由该功能写入 `tool_output_token_limit`
  4. API 卡片编辑入口升级为完整 API 信息编辑，支持修改名称、Base URL、API Key 与模型名称，并同步更新对应 profile
  5. 修复 API 编辑表单高度裁切问题，编辑状态下会完整显示保存与取消操作
  6. 分组展开/收起显示逻辑收口：展开时只显示卡片，收起时只显示条状预览
  7. 分组新增“重排序”操作，可按 5h 重置时间从近到远重排账号，重置时间未知或一致时回到默认排序
  8. 设置页上下文窗口提示补充 `5.5` 模型暂不可用说明
- v1.8.7
  1. 调整分组展开与排序逻辑：收起默认显示条状预览，展开默认显示卡片视图，当前打开的分组会自动置顶
  2. 继续打磨账号页与设置页布局细节，统一分组区与上下文窗口设置区的留白、间距和操作排列
  3. 移除 API 卡片顶部 Provider badge 显示设置，并同步下线该显示功能，减少无效配置项
- v1.8.6
  1. 新增 API 账号 Provider 元数据与标签体系：创建 API 账号时自动识别 Provider，卡片支持展示 Provider 信息并原地编辑标签
  2. 设置页新增后台自动刷新、额度预警与 Provider 顶部标识开关，当前账号余量接近阈值时会给出非打断式提醒
  3. 本地 `/v1` 代理补强账号加模型粒度的 cooldown 退避，支持按 `retry-after`、限流与模型限制结果临时避开失败账号
  4. 修复分组区、卡片区与未分组区的多处布局回归，统一账号卡和 API 卡尺寸，当前账号标识在展开态与收起态都更稳定
  5. 回退分组“一键预热”入口，避免引入批量真预热带来的风控风险
  6. 设置页新增上下文窗口配置，支持 `默认 / 400K / 1M` 预设与手动填写 K 值，并同步写入当前 `config.toml` 与现有账号 profile
- v1.8.5
  1. 分组系统完成重构：移除旧的批量选择式分组交互，改为在分组卡内直接添加未分组账号
  2. 账号页、分组内与未分组区统一采用稳定排序：普通账号默认排在前面，API 条目排在后面，当前账号切换后卡片位置不再跳动
  3. 分组条状视图补回“切换并启动”主操作，并放大名称、标签、额度与重置时间字体，提升可读性
  4. 调整收起态分组卡与顶部工具区宽度对齐，收起后可稳定一排展示 3 个分组
  5. Windows 安装器改为默认直接覆盖安装，不再弹出“卸载并安装 / 直接安装”的选择页
  6. Windows 安装程序完成中文化：NSIS 安装器改用 `SimpChinese`，MSI 安装器新增 `zh-CN` 本地化资源
- v1.8.4
  1. 新增 API 反代三种路由模式：`粘性路由`、`顺序切换`、`混合模式`，分别覆盖按线程固定、按优先级轮询，以及“手动固定线程 + 其余顺序切换”的场景
  2. 新增线程路由面板：可查看最近绑定线程、按 `全部 / 手动 / 自动` 筛选、按线程或账号搜索，并在粘性/混合模式下手动改绑或恢复自动路由
  3. 账号池新增顺序优先级管理，支持手动上下调整，以及按余额自动排序，便于把高优先级账号或指定 API 放在前面
  4. 修复代理账号池快速勾选时可能出现的重复勾选/取消失效问题，减少设置回写较慢时的状态抖动
  5. 调整默认窗口与开发预览尺寸到 `1320 x 960`，启动后可稳定显示 6 张卡片；`npm run dev:desktop` 继续使用隔离 `.dev-runtime` 环境预览
- v1.8.3
  1. 修复删除账号后 `profiles/<account-id>` 缓存目录未同步删除的问题，并在加载账号库时自动清理孤儿 profile 目录
  2. 修复仍可正常读取用量的账号被误显示为“授权过期，请重新登录授权。”的问题
  3. 调整 OAuth 保活刷新策略：预刷新只看 `access_token`，不再因 `id_token` 过期过早触发 refresh；预刷新失败但当前 token 仍可读取用量时，不再立即把账号标记为 blocked
- v1.8.2
  1. 新增“智能切换包含 API（当账号登录无足够限额时）”设置，默认关闭；开启后仅在普通账号余量已明确耗尽时才回退到 API 条目
  2. 修复 API 卡片长接口/模型名溢出问题，并移除重复显示的“重新授权”按钮
  3. 增加隔离开发预览链路：`npm run dev:desktop` 会使用 `.dev-runtime` 中的副本数据启动 Tauri dev，不污染正式安装版账号与 `~/.codex` 配置
- v1.8.1
  1. 回退实验性 NVAPI 导入预设与协议检测入口，导入 API 恢复为 OpenAI 兼容 `/responses` 校验
  2. 移除 `nvapi-...` API Key、接口类型、协议字段与 API 卡片类型/协议显示
  3. 将项目仓库、Issue、Release 与更新检测地址切换到 `Barbital11111/codex-tools`
- v1.8.0

  2. API 条目保存前会进行真实接口检测；检测失败时可查看错误并选择“仍然保存”

- v1.7.8
  1. 基于上游 `v1.7.7`（截至 2026-04-18 的 `upstream/main`）补齐 Windows 下微软商店版 Codex 的启动链路修复
  2. 修复微软商店版 Codex 启动误命中 `Codex Tools` 自身 AUMID 的问题，改为更严格识别 `OpenAI.Codex` 目标
  3. 微软商店版改为优先通过 AUMID 调用系统激活接口，并在返回前校验 Codex 进程已真正拉起；当应用路径启动失败时，再回退到 `codex app`
  4. 自动清理设置中残留的版本化 `WindowsApps\\OpenAI.Codex_<version>...` 路径；即使本地拿不到稳定 exe 路径，也会优先按 AUMID 直启，而不是误提示缺少 Codex CLI
  5. 新增“重新授权账号”入口，可用新的 OAuth 登录态原位替换旧 `authJson`，无需先删除账号再重登
  6. 重新授权时会校验新旧账号身份一致，并同步更新 `accountId`、`planType` 与 usage 快照，修复旧账号长期显示 `free` 的问题
  7. 未选择任何编辑器重启目标时不再额外报错，避免干扰切号后的启动提示
